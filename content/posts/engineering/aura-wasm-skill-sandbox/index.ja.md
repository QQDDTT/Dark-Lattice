# Aura WASM スキルサンドボックス：安全隔離と高性能呼び出しの完璧な融合

![Aura WASM サンドボックス可視化](featured.png)

AI エージェントにコードを実行し、システム API を呼び出す能力を与えることは強力ですが、同時に危険でもあります。Aura は **WebAssembly (WASM)** を使用して、あらゆる実行ノードに対して堅固なスキルサンドボックスを構築しています。

## 1. なぜ Docker ではないのか？

コンテナ技術と比較して、WASM は以下の点で比類のない優位性を持っています。
- **究極の速度**：マイクロ秒単位の起動と破棄。Aura の高頻度なノードジャンプに完璧にマッチします。
- **きめ細かな制御**：各 WASM モジュールがどのホスト API を呼び出せるかを正確に制御できます。

## 2. 安全境界線：デフォルトでゼロ権限

Aura において、各スキルプラグインはデフォルトで**「全盲」**です。
- ネットワークへのアクセス不可。
- ファイルシステムへのアクセス不可。
- 他のプラグインのメモリへのアクセス不可。

Meta が 3D ポインタで明示的に権限を付与し、ホストから対応する「Host Functions」が注入された場合にのみ、そのスキルは相応の権限を持ちます。この**権能ベース（Capability-based）**の安全モデルにより、たとえスキルモジュールにハルシネーションが発生したとしても、その破壊力はサンドボックス内に厳格に制限されます。

## 3. 高性能な ABI 呼び出し

Aura は精錬されたバイナリインターフェースプロトコルを定義しています。Matrix と WASM サンドボックスの間は大容量データの転送に共有メモリを使用し、データシリアル化のオーバーヘッドを大幅に削減しています。これにより、AI エージェントは大規模なテキストストリームを処理する際でも、極めて高いスループットを維持できます。
