# Aura 8 段階モデルルーティング：多次元スコアリングによるコスパ最適解

![Aura モデルルーティング可視化](featured.png)

高価なフラグシップモデルに単純なファイルアーカイブタスクを実行させるのは、リソースの大きな無駄です。Aura は内蔵の **8 段階モデルルーティング（Model Router）**により、各 Matrix ノードに対してきめ細かな計算スケジューリングを実現しています。

## 1. 8 段階の性能階層

利用可能なすべてのモデル（ローカルおよびリモートを含む）を 8 つの階層に分類し、3D アドレッシングの **Model 次元**に対応させます。
- **L1-L3**：ローカルの超軽量モデル。単純な正規表現解析、テキスト分類など、高頻度で低遅延なタスクを担当します。
- **L4-L6**：中量級のリモートモデル。日常的なコードロジック生成や長文要約を担当します。
- **L7-L8**：フラグシップ級の推論モデル。複雑なアーキテクチャ設計、意図分解、衝突解決を担当します。

## 2. 4 次元の動的スコアリング

ルーティングの決定は、各モデルのリアルタイムな 4 次元評価に基づいています。
- **性能（Performance）**：現在のタスクタイプにおけるモデルの歴史的な成功率。
- **遅延（Latency）**：P99 レベルのレスポンスタイム。
- **コスト（Cost）**：トークン単価と予想消費量。
- **安全性（Safety）**：モデルがハルシネーションを生成する統計的確率。

## 3. 自動降格と再ルーティング

高レベルモデルで API 障害やレスポンスタイムアウトが発生した場合、ルーティングエンジンは**「緩やかな降格（Graceful Degradation）」**をトリガーします。Meta はノードポインタの Model 次元を動的に書き換え、タスクを予備のローカルモデルや他のプロバイダーモデルに自動的にリダイレクトすることで、ワークフロー全体の切断を回避します。
