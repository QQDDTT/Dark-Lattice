# Aura 四象限 GC 生命周期：内存锁定与自动释放的平衡

![Aura GC 生命周期可视化](featured.png)

高性能 AI 系统最忌讳的是“垃圾回收（GC）抖动”。为了保证 24-bit 寻址的毫秒级响应，Aura 设计了一套独特的**四象限内存生命周期管理体系**。

## 1. 永久锁定区：系统的“根”

Aura 将核心的 256MB 寻址哈希表和灵魂规则定义为**永久资源**。
这些资源在系统初始化时分配，并使用 Rust 的内存锁定 API 强制驻留在物理内存中，严禁被操作系统交换到磁盘（Swap），确保了 $O(1)$ 寻址的绝对确定性。

## 2. 任务级长期资源

对于一个特定的长程任务，其上下文信息（Context）被定义为**任务级资源**。
这些资源随任务启动而生成，随任务最终归档而销毁。它们存储在 SurrealDB 中，仅在活跃时映射到内存缓存。

## 3. 节点级短期资源

Matrix 执行每一个 24-bit 指针时产生的中间数据属于**节点级资源**。
得益于 Rust 的 **RAII（所有权释放即回收）** 机制，这些资源在 Matrix 节点执行完毕、变量离开作用域的一瞬间，就会被立即物理回收，无需任何背景 GC 扫描。

## 4. WASM 沙箱即用即弃

每一个技能插件都运行在独立的 WASM 实例中。技能执行结束，整个实例连同其占用的线性内存会瞬间被销毁。这种“物理隔离、即用即弃”的模式，从根源上杜绝了 AI 技能插件常见的内存泄漏问题。
