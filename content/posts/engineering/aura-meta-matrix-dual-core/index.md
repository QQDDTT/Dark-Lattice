# Aura 双内核架构：Meta 指挥官与 Matrix 执行者的深度解耦

![Aura 双内核架构可视化](featured.png)

在传统的 AI Agent 设计中，模型往往既负责“思考要做什么”又负责“具体怎么做”。这种职责混淆导致了两个致命问题：**执行过程不可控**以及**长程任务中的意图偏移**。

Aura 通过 **Meta/Matrix 双内核架构** 彻底改写了这一游戏规则。

## 1. Meta 内核：系统的大脑与灵魂

Meta 内核是整个 Aura 系统的“指挥官”。它不直接操作任何文件，也不调用具体技能，其唯一职责是**调度与监控**。

- **意图分析 (S0)**：接收原始需求，利用高等级模型（Level-8）将其解构为抽象的任务拓扑。
- **计划编排 (S1)**：利用**蚁群算法**在 3D 寻址空间中搜索最优路径。
- **结果评估 (S3)**：对 Matrix 上报的产物进行质量判定，并决定是继续推进还是发起 Saga 补偿。

Meta 是系统中唯一拥有全局视角、读取灵魂规则（Soul Rules）并具备“好奇心”的实体。

## 2. Matrix 内核：纯粹的算力与技能池

Matrix 内核是“执行者”。它被设计为一个极度精简、无状态且**绝对被动**的算力引擎。

- **动作执行 (S2)**：接收 Meta 下发的 24-bit 节点指针，加载对应的 WASM 技能插件，输出执行产物。
- **零自主性**：Matrix 禁止自主决定下一个任务节点。它在完成单次执行并将产物推送到 Redis Stream 后，立即进入休眠状态，等待 Meta 的下一次唤醒。

这种“剥夺自由”的设计，确保了系统的安全性——Matrix 永远不会在没有授权的情况下自主修改代码或访问敏感网络。

## 3. 异步连接：Redis Stream 与 ACP 协议

双内核之间通过高度解耦的通信层进行交互：

- **数据池（Data Pool）**：存储任务执行中的中间变量。
- **产物队列（Product Queue）**：Matrix 上报状态的异步通道。
- **ACP 协议**：基于 WebSocket 的全平台通信标准，让控制台（Dashboard）能实时观测到 Meta 与 Matrix 的每一处细微波动。

## 4. 总结：确定性带来的自由

通过 Meta 与 Matrix 的分离，Aura 在底层实现了**确定性执行**，在顶层实现了**自主进化**。Meta 可以尽情探索更优的规划路径，而不用担心底层执行时发生不可预测的崩塌。
