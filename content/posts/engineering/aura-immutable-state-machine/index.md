# Aura 不可逆状态机：单向推进原则与 Saga 补偿模式

![Aura 不可逆状态机可视化](featured.png)

在传统的软件事务中，“回滚（Rollback）”是应对错误的常用手段。但在 AI 代理的世界里，物理回滚往往是不可能的：你无法撤回已经发给客户的邮件，也无法撤回已经写入外部数据库的记录。

Aura 提出了**不可逆状态机（Immutable State Machine）**的设计哲学。

## 1. 状态永远向前

Aura 的状态机被设计为**严格单向**的。一旦一个 Matrix 节点执行成功或失败，系统状态就永久推进到了下一个阶段。
- **禁止回退**：系统不会尝试将数据库恢复到上一个快照来“假装失败没发生”。
- **副作用审计**：每一个状态变迁都伴随着对物理副作用的详细记录，作为后续补偿的依据。

## 2. Saga 补偿模式：在前进中修复

既然不能回头，如何处理错误？Aura 借鉴了分布式事务中的 **Saga 模式**。
当某个节点执行失败时，Meta 会根据当前已产生的副作用，动态生成一个**补偿路径（Compensation Path）**。

- **示例**：如果“上传报告”节点失败，Meta 会调用“删除临时文件”技能作为补偿，然后派生一个新的逻辑分支（比如“通知用户手动上传”），而不是死磕已经失败的上传操作。

## 3. 最终一致性

通过这种设计，Aura 保证了系统的**最终一致性**。它承认世界是不可逆的，并在此基础上构建了一套容错能力极强的执行逻辑，确保系统始终能够回归到一个受控的安全状态。
