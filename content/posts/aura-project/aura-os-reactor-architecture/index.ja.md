# Aura OS 級アーキテクチャのアップグレード：モノリシックカーネルから分散型リアクターへ

![Aura リアクターアーキテクチャの可視化](featured.png)

Aura エージェントが処理するタスクの複雑さが指数関数的に増大するにつれ、従来のモノリシックなプロセスアーキテクチャでは、「リアルタイムなインタラクション」、「長距離の推論」、「ツールの実行」という全く異なるリソース要求をバランスさせることが困難になってきました。この課題を解決するため、Aura は最近、深層的な **OS 級アーキテクチャのアップグレード**を完了し、**Reactor Supervisor（リアクター督導）**を中核とする分散型実行パラダイムを正式に確立しました。

## 1. 核心哲学：装置即プロセス (Reactor as Process)

アップグレード後のアーキテクチャでは、Aura はもはや単一のバイナリ実行体ではありません。各コア能力モジュールは、独立した OS プロセス（「リアクター (Reactor)」と呼ぶ）として疎結合化されています。

- **Pulse (パルス)**：高優先度のユーザーインタラクションとゲートウェイ配信を担当し、いかなる高負荷時でもシステムフィードバックをスムーズに保ちます。
- **Crucible (クルーシブル)**：深層推論と論理演算に特化し、独立した GPU メモリとシステムメモリの割り当て（クォータ）を持ちます。
- **Turbine (タービン)**：物理的なアクション実行（WebHook/PythonHook）を担当し、制限されたサンドボックス環境で動作します。
- **Crystal (クリスタル)**：長距離の進化と知識の固定を管理し、Fisher 情報行列を利用した重み圧縮を行います。

**プロセスレベルの疎結合 (Process-Level Decoupling)** により、Aura は真の障害隔離を実現しました。たとえ特定の推論タスクによって `Crucible` リアクターが OOM（メモリ不足）でクラッシュしたとしても、インタラクションを担当する `Pulse` はユーザーに対して「システムが自己修復中です」と通知し続けることができ、プログラム全体がデッドロックに陥ることはありません。

## 2. ACP 通信：IPC ベースのシグナルバス

プロセス疎結合化による課題は通信コストです。Aura は不安定な内部チャネルをすべて廃止し、**ACP (Aura Control Protocol)** に基づく高性能な IPC メカニズムに全面的に移行しました。

### 2.1 プロトコルスタックの選択
コアリンクとして、**Unix Domain Sockets (UDS)** と **Postcard** シリアライズ形式を採用しました。極限の状態では、Zenoh などの分散メッセージバスにもシームレスに拡張可能です。この設計により、プロセス間通信の遅延をマイクロ秒単位に抑え、分散アーキテクチャ下でもパフォーマンス損失をほぼゼロにしています。

### 2.2 シグナル中継と自己修復
**Aura Kernel** は、この時「督導器」と「スイッチ」の二重の役割を果たします。ACP メッセージの中心的な中継と配信を行うだけでなく、PID 監視とハートビートメカニズムを通じて各リアクターの状態をリアルタイムで感知します。リアクターの異常を検知すると、カーネルは直ちに子プロセスの再起動を実行し、`CausalSnapshot` を利用してコンテキストの連続性を回復します。

## 3. リソース統治：熱力学スケジューリングモデル

マルチプロセス環境で精密なリソーススケジューリングを実現するため、**Thermodynamic Scheduler（熱力学スケジューラ）**を導入しました。

このスケジューラは、単に CPU 使用率に基づいてリソースを割り当てるのではなく、2 つの核心的な物理指標を導入しています。
1. **エントロピー (Entropy)**：システムの無秩序度を測定します。あるリアクターのエントロピーが高すぎる場合、スケジューラは保護的な拒絶（Protection Refusal）を発動します。
2. **変分自由エネルギー (VFE)**：予測と現実の乖離を測定します。高い VFE は動的なレート制限を誘発し、システムを「冷却期間」へと導いて認知を整合させます。

この物理法則に基づいたスケジューリングモデルにより、突発的なトラフィックに直面した際の Aura は、硬直したソフトウェアではなく、生命体のように振る舞います。

## 4. 安全防護：ACP 認可フロー

分散環境下では、セキュリティの境界がより明確になります。Aura は **ACP 認可フロー (Authorization Flow)** を実装しました：
- 機密性の高いアクション（ファイルの削除、外部 API の呼び出しなど）はカーネルによって傍受されます。
- 傍受された指令はステージングエリアに入り、ダッシュボード経由で手動認可を要求します。
- この「カーネル級の傍受」メカニズムは、AI の行動安全に対する最後の防衛線となります。

## 5. まとめ

今回の OS 級アーキテクチャのアップグレードは、Aura が「複雑なインテリジェンスエンジン」から「堅牢な AI オペレーティングシステム」へと脱皮したことを意味します。プロセスレベルの疎結合、高性能 IPC、そして革新的な熱力学スケジューリングにより、Aura は極限のシステム安定性を手に入れただけでなく、将来の分散型協調と長距離の進化に向けた強固な技術基盤を築きました。
